CubeRTOS - Custom Real Time Operating System

CubeRTOS is a lightweight, custom-built real-time operating system that I designed and implemented for embedded flight and payload systems (Primarily Cube Satellites). It was developed to provide deterministic task scheduling, robust fault handling, and low-overhead operation on resource-constrained hardware where existing RTOS solutions were either too heavy or insufficiently transparent.

At its core, CubeRTOS features a real-time scheduler designed to meet strict timing constraints while maintaining predictable execution behavior. I implemented task management and timing mechanisms entirely in C, with careful attention to interrupt safety, memory usage, and execution latency. The system was built to be highly modular, allowing tasks to be added or reconfigured with minimal impact on system stability through JSON configuration files.

A key focus of CubeRTOS was fault detection and system reliability. I designed monitoring mechanisms to detect task overruns, missed deadlines, and unexpected system states, enabling the software to respond gracefully to faults rather than failing silently. This was particularly important for aerospace and embedded applications where reliability and debuggability are critical.

CubeRTOS was successfully used to support sensor management, data acquisition, and communication tasks, integrating tightly with low-level drivers and telemetry systems. Through this project, I gained hands-on experience with real-time systems design, scheduler theory, and hardware–software integration, while building an RTOS from the ground up rather than treating it as a black box.

Technologies & Concepts: C/C++, real-time scheduling, embedded systems, interrupt-driven design, fault detection, low-level hardware interaction, deterministic software design
