CubeSat Flight Laboratory - Alpha Team

Currently serving as the hardware and software lead of a small engineering team developing a CubeSat designed to detect Forbush decreases in the atmosphere. The system uses a Raspberry Pi Zero running Linux as the primary flight computer, interfaced with multiple auxiliary sensors via Qwiic connectors. Target launch date is late November of this year. (System pictured above.)

CubeRTOS

Currently developing a lightweight, memory-efficient RTOS in C for our CubeSat mission. Implemented software multi-threading to run tasks before, during, and after flight operations. Designed a custom scheduler in C that executes user-defined tasks across four worker threads while using a dedicated scheduling thread to ensure all tasks meet timing requirements.

AstroCam

Currently serving as Chief Engineer for the University of Michigan’s Bioastronautics and Life Support Systems (BLiSS) AstroCam project, leading the design, manufacturing, testing, and verification of a lightweight camera for use in microgravity environments. Utilizing Altium Designer to create and verify our PCBs.

  • Directing design and development of a dual-PCB hand held camera for use in micro-gravity within a $10k budget using MBSE principles.
  • Calculated worst-case power draw for PCB system to optimize the design, reducing weight by 10 while maintaning required mission lifetime.
  • Currently directing fabrication, testing, and design verification of system.

Autonomous Parafoil

Currently serving as Chief Engineer for an advanced embedded systems project developing an Autonomous Parafoil as a low-cost recovery method for in-atmosphere scientific testing. The system, now in the planning phase with a functional demo targeted for late November, will feature a ground station capable of sending GPS landing coordinates to a custom PCB-based autonomous payload. The payload will communicate via DigiMesh over XBee modules and utilize an Extended Kalman Filter with a suite of sensors to guide itself to a landing site with ~100 m accuracy.

Ping Pong Robot

Led a team of 4 engineers in designing and verifying a 2-axis ping pong robot capable of rallying with a player for 30 seconds. Implemented three operating modes: (1) Wiimote control using a custom Bluetooth interface, (2) PS2 controller mode for manual paddle movement and swinging, and (3) autonomous mode with a Raspberry Pi 4 running OpenCV for real-time ball tracking. Integrated SPI, I²C, voltage regulation, and STM32 microcontroller for system control.

Technical Responsiblities

  • Designed, manufactured, and tested the mechanical system using stepper motors, drive belts, and 80/20 aluminum extrusion.
  • Developed custom stepper motor drivers with built in damping to acceleration and deceleration to streamline integration with the main control program.
  • Led testing campaigns across all three operating modes to ensure reliable system performance.
  • Assembled and soldered three stepper motor driver circuits on protoboards.

Bloom Box - Bioastronautics and Life Support Systems

Led a team of four engineers through the Model-Based Systems Engineering (MBSE) design cycle to develop a CO2 bioregenerative algae scrubber for a Collins Aerospace–funded project. This was my first experience serving as the lead for both electronics and software. The system needed to perform several mission-critical tasks periodically, including pH measurement of the algae, water and nutrient distribution, tank cleaning, and real-time information display. To achieve this, we implemented a distributed control system using five Arduino Unos as worker nodes, managed by a central Raspberry Pi acting as the data collector and task distributor that were dasiy-chained together to communicate using SPI.
