IonEngine - Custom Unity-like Game Engine

IonEngine is an open-source game engine I designed in a single semester inspired by the Unity Game Engine. It features a custom editor built using the Dear ImGui library designed to feel familiar to Unity developers, a custom data-oriented particle systems, a physics engine utilizing Box2D, scripting capabilities using Lua, and more.

Engine Stack

  • C++ - The backbone of both the runtime engine and the editor, delivering high performance and control.
  • LuaBridge - A lightweight library that binds C++ functions and classes to Lua. Implemented to make scripting sealess and intuitive.
  • SDL - Cross-platform library used to control windowing, input, audio, and rendering on Windows, OSX, and Linux.
  • Box2D - The physics engine that controls colliders, triggers, and rigidbody physics.
  • Dear ImGui - Immediate mode GUI used to build IonEngine's custom in-editor interface.
  • RapidJSON - All information regarding scenes, actors, and components are stored in JSON files that need to be read/written to quickly.

I was the sole developer working on this engine with weekly check-ins on progress. Starting with a text adventure game, I rapidly evolved IonEngine to include 2D rendering and audio output using the SDL library, allow scripting through Lua and the LuaBridge library, integrate a full editor to provide ease of development, and utilize rigidbody physics to simulate movement.

I also implemented a custom Data-oriented particle system that was fully written in C++ and accessable in Lua through the LuaBridge library. This was to allow millions of particles to be rendered on screen and simulated with minimal losses in performance.
